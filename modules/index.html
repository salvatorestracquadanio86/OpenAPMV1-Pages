Platform Modules

Six focused modules, designed to work independently and grow together — all centred on the asset.

Design Philosophy

Everything starts from the asset

OpenAPM is an asset-centric system. Every event, every intervention, every document, every measurement is traceable back to a specific asset — located in a precise place, with its own history.

✅ Implemented
OAH

Operational Asset Hub

The heart of the system — included in every configuration.

OAH manages the location hierarchy (plants, departments, areas, units — freely modellable for any industry), the equipment registry with type-specific technical characteristics, and the universal event timeline.

Everything that happens on an asset — a failure, an intervention, an inspection, an anomaly — is recorded chronologically. This register is the memory of the asset.

🗺️

Location Hierarchy

Flexible multi-level structure adaptable to any industry or site layout

⚙️

Equipment Registry

Centralised asset catalog with dynamic technical specs per asset type

📅

Event Timeline

Chronological record of every event on every asset — the plant's long-term memory

Functional Modules

Built on top of OAH

Each module extends OAH with a specific operational process — and connects seamlessly with the others.

✅ Implemented
🔧
WOM

Work Order Management

Covers the full operational maintenance cycle — from field observations to formal work orders, intervention reports, and downtime tracking.

  • Field maintenance notes & anomaly reports
  • Full work order lifecycle with approval workflow
  • Contractor & internal execution management
  • Downtime tracking → MTBF / MTTR source data
  • ERP integration ready (Odoo, SAP, custom)
Note → Work Order → Report → OAH Timeline
🔧 In Progress
🔍
INS

Inspection Module

Manages one-shot inspections with adaptive forms that change based on the inspection type — guiding the user without requiring specialist knowledge.

  • Visual, checklist, thermographic, vibration
  • Ultrasonic thickness (UT), pressure testing
  • Regulatory compliance & safety audits
  • Adaptive form per inspection type
  • Auto-generates WO on negative result
Plan → Execute → Result → WO if FAIL
📐 Design
📆
SCHED

Scheduling & Planning

Introduces recurring planning for preventive maintenance and inspections — with pre-built templates for common asset types so no one starts from zero.

  • Preventive maintenance & inspection plans
  • Pre-seeded templates per asset type
  • Time-based, counter-based, condition-based triggers
  • Occurrence tracking & compliance rate KPIs
  • Maintenance budget forecast
Plan → Occurrence → WO / INS → KPI
📐 Design
📊
AIM

Asset Integrity Monitoring

Transforms data into knowledge across three pillars: risk assessment, condition monitoring, and reliability statistics — all feeding back into OAH.

  • Guided risk matrix wizard (P × C, 5×5)
  • KPI monitoring with 3-zone thresholds
  • Auto health status update on OAH
  • MTBF, MTTR, Availability — auto-calculated
  • Failure Pareto & reliability trend dashboard
KPI Data → Health Score → Risk Level → OAH
📐 Design
📓
LOG

Digital Logbook

Replaces paper sheets and chat groups with a structured operational register — contextualised to assets, with a built-in shift handover workflow.

  • Operational notes on assets or locations
  • Shift handover workflow (outgoing → incoming)
  • One-click escalation to WOM
  • Full-text search & shift report export
  • Included in the free baseline tier
Note → Handover → Escalate → WOM Note

Integration

How the modules connect

These flows are the natural behaviour of modules designed to work together from the very beginning.

⚡

Sensor alert to work order

AIM detects KPI threshold breach → OAH health status updates → WOM work order generated → AIM reliability stats updated on closure
🔄

Shift handover to intervention

Operator logs anomaly in LOG → Shift leader escalates to WOM → Technician executes & closes WO → OAH timeline updated automatically
📋

Inspection plan to corrective action

SCHED triggers annual inspection → INS inspection executed — FAIL result → WOM corrective WO auto-created → AIM risk assessment updated
Design Principles

Three principles behind every choice

🎯

Guide, don't just collect

At every critical point there is a mechanism that helps the user — wizards, adaptive forms, templates, checklists. The goal is not a database of records, but helping people do things well without needing to be experts.

🌱

Start from what exists

No organisation starts from zero. OpenAPM is designed to gradually incorporate existing reality — not to replace it with a system imposed from above. You can start with one module and grow.

📌

Traceability as value

Every action generates a trace — not to monitor people, but to build the memory of the plant. That memory, over time, becomes the most valuable asset of any operational organisation.
