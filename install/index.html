Install OpenAPM

Distributed as a Docker image. You only need Docker installed — no build steps, no dependencies.

Prerequisites

Docker 24+ with Docker Compose. Nothing else.

1 — Pull the image

Pull the latest beta image directly from Docker Hub:

docker pull openapmproject/openapm:0.1.0-beta.1

2 — Download the compose file

Download the docker-compose.yml file and place it in a local folder of your choice. The stack includes MongoDB 7 — no external database required.

⬇ Download docker-compose.yml

Or copy the contents below:

# docker-compose.yml — OpenAPM (OAH + WOM modules, MongoDB included)
#
# ── Quick start ───────────────────────────────────────────────
#   docker compose up -d
#
# ── First-time DB initialisation (required) ───────────────────
#   docker compose exec openapm node scripts/seed-setup.js
#
# ── Demo data (optional) ──────────────────────────────────────
#   docker compose exec openapm node scripts/seed-demo-oah.js
#   docker compose exec openapm node scripts/seed-demo-work-orders.js
#   docker compose exec openapm node scripts/seed-demo-ins-inspections.js
#
# ── Admin password ────────────────────────────────────────────
#   Printed to stdout on first boot, or retrieve it with:
#   docker compose exec openapm cat /data/admin_password.txt
#
# ── Application ───────────────────────────────────────────────
#   Once running, open http://localhost:4500/ui in your browser.
#   Swagger API docs: http://localhost:4500/api

services:

  mongo:
    image: mongo:7
    restart: unless-stopped
    volumes:
      - mongo_data:/data/db
    environment:
      MONGO_INITDB_DATABASE: openapm
    healthcheck:
      test: ["CMD", "mongosh", "--eval", "db.runCommand({ ping: 1 })"]
      interval: 10s
      timeout: 5s
      retries: 5
      start_period: 20s

  openapm:
    image: openapmproject/openapm:beta
    restart: unless-stopped
    ports:
      - "4500:4500"
    depends_on:
      mongo:
        condition: service_healthy
    volumes:
      - openapm_data:/data
    environment:
      NODE_ENV: production
      PORT: 4500
      MONGODB_CONNECTION_STRING: mongodb://mongo:27017
      MONGODB_NAME: openapm
      # IMPORTANT: replace with a long random string before going to production
      # Generate with: openssl rand -base64 48
      JWT_SECRET: change-this-secret-in-production
      JWT_EXPIRATION: 8h
      SML_ADMIN_USER: admin
      SML_ADMIN_PWD: changeme
      SWAGGER_ENABLED: "true"

volumes:
  mongo_data:
  openapm_data:

3 — Start the stack

docker compose up -d

This starts both the OpenAPM application and MongoDB. MongoDB is health-checked before OpenAPM starts — no race conditions.

4 — Initialise the database

Run this once after the first boot. It creates all required collections, system codes, UI views, and default roles.

docker compose exec openapm node scripts/seed-setup.js

5 — Open the application

Navigate to http://localhost:4500/ui in your browser.

Log in with:

  • Username: admin
  • Password: printed to the container log on first boot, or retrieve it with:
docker compose exec openapm cat /data/admin_password.txt

⚠️ Change the admin password immediately after first login.

Swagger API docs are available at http://localhost:4500/api

Optional — Load demo data

Explore the application with realistic sample data across all available modules:

docker compose exec openapm node scripts/seed-demo-oah.js
docker compose exec openapm node scripts/seed-demo-work-orders.js
docker compose exec openapm node scripts/seed-demo-ins-inspections.js

Configuration

All configuration is passed via environment variables in docker-compose.yml.

Variable Default Description
MONGODB_CONNECTION_STRINGmongodb://mongo:27017MongoDB URI
MONGODB_NAMEopenapmDatabase name
JWT_SECRET(placeholder)Change this — use a long random string
JWT_EXPIRATION8hSession token lifetime
SML_ADMIN_USERadminDefault admin username
SML_ADMIN_PWDchangemeDefault admin password — change after first login
SWAGGER_ENABLEDtrueExpose API docs at /api

To generate a secure JWT_SECRET:

openssl rand -base64 48

Updating to a new version

docker compose pull
docker compose up -d
